OWASP API Security Top 10 (2023)
10 automated test modules covering every category
Each module implements targeted detection techniques for its OWASP category, with severity classification, detailed findings, and actionable remediation guidance.
API1
Broken Object Level Authorization
Tests for IDOR vulnerabilities by manipulating object identifiers. Detects when resources belonging to other users are accessible by incrementing numeric IDs, substituting UUIDs, or replacing MongoDB ObjectIDs.
Techniques: ID increment, UUID swap, ObjectID brute, response comparison
API2
Broken Authentication
Probes authentication mechanisms for weaknesses. Tests weak credential pairs against login endpoints, checks for missing authentication on sensitive paths, and detects token leakage in responses.
Techniques: Credential stuffing (5 pairs), auth bypass, token detection
API3
Broken Object Property Level Authorization
Identifies mass assignment and excessive data exposure. Scans API responses for 16 sensitive field patterns (password, token, api_key, ssn, role) and tests writable endpoints for privilege escalation via property injection.
Techniques: 16 field signatures, mass assignment payloads, role injection
API4
Unrestricted Resource Consumption
Validates rate limiting and resource controls. Sends 20 rapid requests to detect missing 429 responses, checks for rate limit headers, tests 100KB payload acceptance, and probes pagination bypass with limit=999999.
Techniques: Burst requests, header check, payload size, pagination abuse
API5
Broken Function Level Authorization
Discovers accessible administrative and debug endpoints. Probes 25 common paths (/admin, /actuator, /debug, /graphql, /metrics) and tests HTTP method tampering (sending DELETE to GET-only endpoints).
Techniques: 25 admin paths, method tampering (GET/PUT/DELETE/PATCH)
API6
Unrestricted Access to Sensitive Flows
Detects business-critical operations lacking anti-automation controls. Identifies purchase, registration, password reset, and OTP endpoints, then checks for CAPTCHA, CSRF tokens, or step-up authentication.
Techniques: 16 sensitive patterns, CAPTCHA/CSRF detection
API7
Server Side Request Forgery (SSRF)
Tests URL-accepting parameters for internal network access. Injects internal targets (127.0.0.1, AWS IMDS 169.254.169.254, GCP metadata, Azure IMDS) through 16 common URL parameter names and checks for cloud metadata indicators.
Techniques: 16 URL params x 3 internal targets, IMDS detection
API8
Security Misconfiguration
Audits security posture across CORS, headers, error handling, and HTTP methods. Tests origin reflection, checks 6 security headers, detects server version disclosure, probes for verbose stack traces, and identifies TRACE/TRACK methods.
Techniques: CORS reflection, 6 headers, error probing, method audit
API9
Improper Inventory Management
Discovers exposed documentation, debug, and legacy endpoints. Concurrently scans 50+ paths including Swagger/OpenAPI, GraphQL, Spring Boot Actuator, .git, .env, old API versions (/v1, /v2, /beta, /staging, /dev), and pprof debug.
Techniques: 50+ path brute, concurrent scanning, category classification
API10
Unsafe Consumption of APIs
Tests for injection vulnerabilities through API inputs. Sends payloads for 6 injection categories via both query parameters and JSON body: SQL (4 payloads), NoSQL (3), XSS (2), SSTI (3), command injection (4), and path traversal (2).
Techniques: SQL, NoSQL, XSS, SSTI, cmd injection, path traversal
Deployment
Quick Start
APIShield requires Python 3.8+ and the requests library. No API keys, accounts, or cloud services required. Runs entirely on your infrastructure.
# Install git clone https://github.com/SiteQ8/APIShield.git cd APIShield && pip install requests # Full scan against a target API python3 apishield.py -u https://api.example.com # Authenticated scan with Bearer token python3 apishield.py -u https://api.example.com -H "Authorization: Bearer TOKEN" # Scan specific endpoints python3 apishield.py -u https://api.example.com -e /users,/orders,/admin/config # Run only specific OWASP tests python3 apishield.py -u https://api.example.com --tests API1,API4,API8 # Skip TLS verification (internal/staging APIs) python3 apishield.py -u https://staging-api.internal --no-verify
Reference
Command-Line Parameters
ParameterTypeDescription
-u, --urlRequiredBase URL of the target API
-e, --endpointsOptionalComma-separated endpoint paths to test (e.g., /users,/orders)
-H, --headerOptionalCustom HTTP header — repeatable (e.g., "Authorization: Bearer token")
--testsOptionalSpecific OWASP tests to run (e.g., API1,API4,API8)
--timeoutOptionalHTTP request timeout in seconds (default: 10)
--no-verifyOptionalSkip TLS certificate verification for internal APIs
--versionFlagDisplay version number and exit
Output
Reporting
Every scan automatically generates two report formats. Both include per-finding severity classification, risk description, and actionable remediation guidance.
JSON
Structured data with full metadata, severity, findings, and remediation. Designed for CI/CD pipeline integration, SIEM ingestion, and GRC platform import.
HTML
Visual report with color-coded severity, compliance score, and detailed findings. Suitable for management review, audit evidence, and stakeholder communication.
